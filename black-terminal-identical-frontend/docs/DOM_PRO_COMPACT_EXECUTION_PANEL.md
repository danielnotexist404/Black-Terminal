# DOM Pro Compact Execution Panel

Status: Implemented on the existing execution path

## Contract

The compact DOM ticket is a presentation of `VenueExecutionSchema`. It does not create an alternate broker path.

Submission remains:

DOM Pro ticket -> `submitOrder` -> OMS -> EMS -> Risk -> Broker Router -> venue adapter

The panel synchronizes the selected Bybit account through the existing account sync endpoint and uses the same schema, instrument rules and order preview functions as Unified Execution Ticket.

## Controls

- Order Type is a compact selector. Only modes with a complete compact-ticket implementation are exposed.
- Limit orders expose venue-supported GTC, IOC and FOK. Market orders display venue-default TIF. Post-Only restricts TIF to GTC.
- Margin mode and leverage are separate controls. Leverage options are bounded by instrument metadata and the active risk tier.
- Equity Allocation uses markers at 0, 1, 5, 10, 15, 25, 35, 50, 65, 75 and 100 percent.
- Quantity, order value, required margin, entry fee and balance after are generated by the existing venue sizing/preview engine.
- Post-Only, Reduce-Only and TP/SL retain venue validation.

Equity Allocation sizes from usable account margin after fee and leverage effects. It does not equate total equity with executable notional. Changing leverage updates margin independently and recalculates an active percentage allocation without silently changing account mode.

## Responsive Modes

Container width controls the presentation. Wide panels use one dense control row; medium and narrow panels use progressively smaller grids and remove secondary labels before controls can overlap. All values remain unscaled text and native form controls.
